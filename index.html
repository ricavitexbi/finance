🗄️ Estrutura do Banco de Dados

Importante: Se você está recebendo erro relacionado a 'user_id', use a estrutura SIMPLES abaixo:
Opção 1 - Estrutura SIMPLES (recomendada para começar):
CREATE TABLE transactions (
    id SERIAL PRIMARY KEY,
    data DATE NOT NULL,
    tipo VARCHAR(10) NOT NULL,
    categoria VARCHAR(50) NOT NULL,
    valor DECIMAL(10,2) NOT NULL,
    descricao TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Opcional: Criar índice para melhor performance
CREATE INDEX idx_transactions_data ON transactions(data DESC);
Opção 2 - Estrutura COMPLETA (com isolamento por usuário):
CREATE TABLE transactions (
    id SERIAL PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id),
    data DATE NOT NULL,
    tipo VARCHAR(10) NOT NULL,
    categoria VARCHAR(50) NOT NULL,
    valor DECIMAL(10,2) NOT NULL,
    descricao TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Ativar RLS
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;

-- Criar política
CREATE POLICY "Users can only see own transactions" 
ON transactions FOR ALL 
USING (auth.uid() = user_id);
